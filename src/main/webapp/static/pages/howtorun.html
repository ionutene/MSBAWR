To run automated tests properly, be sure the next steps are done:

1. Automated tests code

1.1 If you made any changes in tests code do the next steps , else skip entire step 1
  • Commit your code on SVN
  • Go on Jenkins Project and run a build or release (or wait 1 hour from the last commit, and auto build will start :) )
    • If you created a release you have to Promote it as "Approved" and create one more build after that
  • Click on the REINDEX button (You must selected no environment)

2. MSBAdapter release version

2.1 If the desired MSBAdapter version is already installed on the environment, skip entire step 2
2.2 Be sure that the MSBAdapter release on Jenkins Project that you want to test has:
  • Promotion status "QA Ready" - if not promote it
  • One more build is created after promoted release - if you just promote a release, create a new build

3. Run automated tests

3.1 Select one environment - only tests for that environment will be available
3.2 Select the desired tests
3.3 Click Run :)

NOTE: We recommend to run first the Installation tests (ALONE) and wait if the correct MSBAdapter version was successfully installed. After, un-check installation test and run the others tests (It is possible that installation test fail. This means all the others tests are useless ). If you installed manually the the MSBAdapter version on the environment skip "installation test"
